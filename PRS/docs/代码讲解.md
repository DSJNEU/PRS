# PRS 

本项目的代码围绕三条主线组织：流水线模板容错、MoE 专家适配、预测式资源调度。项目分成两层：一层是可在普通开发机上运行的演示闭环，另一层是接入 PyTorch/torchrun 的真实训练实验。

## 流水线模板容错

对应代码：

- `prs/planning/__init__.py`
- `prs/engine/__init__.py`

核心逻辑：

- `LayerProfile` 记录每层的前向时间、反向时间、显存和参数量。
- `PipelinePlanner.create_template()` 用动态规划切分连续层，目标是降低最慢阶段的时延。
- `PipelineTemplate` 保存阶段数、层分配、阶段时延、阶段显存和节点映射。
- `PRSEngine.reconfigure()` 在故障后根据剩余节点数选择可用模板，并从检查点恢复训练状态。

方法对应：

- 显存约束对应 `segment_memory(split, end) <= gpu_memory`。
- 阶段均衡目标对应 `worst_time = max(dp[split][stage - 1], stage_time)`。
- 模板库由 `pipeline_templates` 维护，故障恢复时按可用节点数查表。

## 检查点恢复

对应代码：

- `prs/checkpoint/__init__.py`

核心逻辑：

- `IncrementalCheckpoint.diff()` 递归比较状态，只保存变化项。
- `IncrementalCheckpoint.merge()` 从完整检查点和增量差异重建状态。
- `HierarchicalCheckpointManager.classify_state()` 按 key 将训练状态分成热状态和冷状态。
- `CheckpointManager` 负责元数据、保留策略、异步保存和恢复。

真实训练中，`TorchExperimentRunner` 会把模型参数、优化器状态、训练状态和实验配置交给 `CheckpointManager` 保存。

## 弹性故障检测与恢复

对应代码：

- `prs/elastic/__init__.py`
- `prs/engine/__init__.py`
- `prs/experiments/torch_runner.py`

核心逻辑：

- `HeartbeatMonitor` 维护节点心跳，并根据近期延迟计算超时阈值。
- `FailureDetector.detect()` 生成 `ReconfigurationEvent`。
- `ElasticController.mark_failed()` 用于实验中的故障注入和恢复触发。
- `TorchTrainingConfig.fault` 配置故障步数、rank 和故障模式。

真实实验可用 `--fault-step`、`--fault-rank`、`--fault-mode exit` 让指定 rank 退出；重新使用同一个输出目录运行时，PRS 会读取已有检查点继续训练。

## MoE 专家适配

对应代码：

- `prs/moe/__init__.py`
- `prs/engine/__init__.py`
- `prs/experiments/torch_runner.py`

核心逻辑：

- `LoadHeatmapMonitor` 统计每层每个专家的 token 数。
- `FaultAwareReplicaAllocator` 按 token 占比分配专家副本，并保留容错下限。
- `MROReplicaPlacer` 将专家分组后映射到节点池。
- `ZeroPaddingCommunicator` 统计本地 token、远程 token 和 padding 节省量。

训练入口可以传入 MoE 路由统计；示例会使用稳定的路由分布来验证适配流程。

## 预测式主动调度

对应代码：

- `prs/scheduler/__init__.py`

核心逻辑：

- `ARIMANodePredictor` 用节点历史序列预测后续可用节点数。
- `LiveputCalculator.compute_liveput()` 估计有效训练收益。
- `DPConfigSearcher.search()` 在预测窗口内搜索并行配置路径。
- `PredictiveResourceScheduler.step()` 输出调度决策，并记录动作。

方法对应：

- 候选配置来自 `candidate_configs()`。
- 状态转移对应 `prev_value + liveput - migration_cost`。
- 最优路径通过回溯 `path.reverse()` 得到。

## 真实训练实验层

对应代码：

- `prs/experiments/torch_runner.py`
- `prs/experiments/metrics.py`
- `prs/experiments/report.py`
- `examples/run_real_training.py`

能力范围：

- 使用内置小型 causal LM 做烟测。
- 加载 HuggingFace `AutoModelForCausalLM`。
- 通过 `torchrun` 跑单机多卡或多机多卡 DDP。
- 每步记录 loss、吞吐、耗时和模板 ID。
- 支持故障注入、检查点恢复和周期性调度。
- 输出 Markdown 和 JSON 两种实验记录。

## 演示层

对应代码：

- `examples/run_demo.py`
- `prs/web/monitor.py`
- `prs/web/templates/index.html`

用途：

- 在没有 GPU 和 PyTorch 的机器上验证 PRS 控制闭环。
- 通过 Web 页面观察训练状态、节点状态、模板切换、检查点、MoE 和调度决策。

## 复现实验的边界

具体数值依赖硬件、驱动、CUDA/NCCL、模型规模和 baseline 配置。复现 V100、Qwen1.5-MoE-A2.7B 等实验时，需要在对应集群上固定软件版本、故障注入间隔和训练参数。
