# 单机 8GPU 最小跑通测试



```bash
cd /home/dsj/PRS
source .venv-linux/bin/activate

CUDA_VISIBLE_DEVICES=0,1,2,3,4,5,6,7 \
NCCL_DEBUG=WARN \
TORCH_NCCL_BLOCKING_WAIT=1 \
torchrun --standalone --nproc_per_node=8 -m prs.experiments.cli \
  --output-dir .prs_runtime/server_8gpu_example_final \
  --model-source synthetic \
  --steps 4 \
  --batch-size 1 \
  --seq-len 64 \
  --hidden-size 128 \
  --layers 8 \
  --heads 4 \
  --checkpoint-interval 2 \
  --scheduler-interval-steps 2
```

参数含义：

```text
CUDA_VISIBLE_DEVICES=0-7       使用 8 张 V100
torchrun --standalone          单机分布式模式
--nproc_per_node=8             启动 8 个 DDP 进程
--model-source synthetic       使用项目内置 synthetic 小模型/假数据
--steps 4                      只跑 4 步，快速验证
--checkpoint-interval 2        第 2、4 步保存 checkpoint
--scheduler-interval-steps 2   第 2、4 步触发调度逻辑
```

文档中的 8GPU 实验退出码为 0，并生成 `summary.json`、`experiment_report.md`、`experiment_report.json` 等报告文件。

运行时另开一个终端查看 GPU：

```bash
watch -n 1 nvidia-smi
```

如果看到 8 张 GPU 都有 Python 进程，说明 8 卡已经启动。

查看 8GPU 结果

```bash
cat .prs_runtime/server_8gpu_example_final/summary.json
```

查看报告：

```bash
cat .prs_runtime/server_8gpu_example_final/experiment_report.md
```

查看输出文件：

```bash
find .prs_runtime/server_8gpu_example_final -maxdepth 3 -type f | sort
```

检查训练结束后是否有 GPU 残留进程：

```bash
nvidia-smi --query-compute-apps=pid,process_name,used_memory --format=csv,noheader
```

如果输出为空，说明没有残留训练进程。

# 五分钟容错训练实验

五分钟实验分成两个阶段：

```text
第一阶段：运行 8GPU 训练，在第 50 步注入 rank0 故障。
第二阶段：使用同一个输出目录重启训练，从 checkpoint 恢复到 step=49 附近并继续到 step=300。
```

------

## 第一阶段：注入故障

```bash
cd /home/dsj/PRS
source .venv-linux/bin/activate

/usr/bin/time -p env \
CUDA_VISIBLE_DEVICES=0,1,2,3,4,5,6,7 \
NCCL_DEBUG=WARN \
NCCL_SOCKET_IFNAME=lo \
TORCH_NCCL_BLOCKING_WAIT=1 \
.venv-linux/bin/torchrun \
  --nproc_per_node=8 \
  --max_restarts=1 \
  --master_addr=127.0.0.1 \
  --master_port=29617 \
  -m prs.experiments.cli \
  --output-dir .prs_runtime/five_min_fault_demo_stable \
  --model-source synthetic \
  --steps 300 \
  --batch-size 1 \
  --seq-len 256 \
  --hidden-size 512 \
  --layers 8 \
  --heads 8 \
  --checkpoint-interval 5 \
  --scheduler-interval-steps 25 \
  --fault-step 50 \
  --fault-rank 0 \
  --fault-mode exit
```

这个命令会在第 50 步附近让 rank0 退出，模拟故障。

第一阶段结束后检查：

```bash
tail -n 12 .prs_runtime/five_min_fault_demo_stable/metrics_rank0.jsonl
find .prs_runtime/five_min_fault_demo_stable/checkpoints -maxdepth 2 -type f | sort | tail -n 20
test -f .prs_runtime/five_min_fault_demo_stable/fault_step50_rank0.done && echo fault_marker_exists
```

如果看到：

```text
fault_marker_exists
```

说明故障标记文件已经生成。

------

## 第二阶段：从 checkpoint 恢复

第二阶段继续使用同一个输出目录：

```bash
cd /home/dsj/PRS
source .venv-linux/bin/activate

/usr/bin/time -p env \
CUDA_VISIBLE_DEVICES=0,1,2,3,4,5,6,7 \
NCCL_DEBUG=WARN \
NCCL_SOCKET_IFNAME=lo \
TORCH_NCCL_BLOCKING_WAIT=1 \
.venv-linux/bin/torchrun \
  --nproc_per_node=8 \
  --master_addr=127.0.0.1 \
  --master_port=29620 \
  -m prs.experiments.cli \
  --output-dir .prs_runtime/five_min_fault_demo_stable \
  --model-source synthetic \
  --steps 300 \
  --batch-size 1 \
  --seq-len 256 \
  --hidden-size 512 \
  --layers 8 \
  --heads 8 \
  --checkpoint-interval 5 \
  --scheduler-interval-steps 25 \
  --fault-step 50 \
  --fault-rank 0 \
  --fault-mode exit
```

因为第一阶段已经生成：

```text
fault_step50_rank0.done
```

所以第二阶段不会重复注入故障，而是加载已有 checkpoint 恢复训练。

# 常见问题处理

## 1. `ModuleNotFoundError: No module named 'sympy'`

说明虚拟环境缺包：

```bash
source .venv-linux/bin/activate
python -m pip install sympy
```

------

## 2. `python: command not found`

说明虚拟环境没激活，或者 venv 坏了。

先执行：

```bash
cd /home/dsj/PRS
source .venv-linux/bin/activate
which python
```

如果没有：

```bash
/home/dsj/PRS/.venv-linux/bin/python
```

就重建：

```bash
rm -rf .venv-linux
python3 -m venv .venv-linux
source .venv-linux/bin/activate
```

------

## 3. `getcwd() failed: No such file or directory`

说明当前所在目录已经不存在。先跳出去：

```bash
cd /
```

再进入项目：

```bash
cd /home/dsj/PRS
```

如果项目不存在，重新上传源码。

------

## 4. `Address already in use`

Web 端口被占用。

查：

```bash
sudo ss -ltnp | grep ':5100'
```

杀进程：

```bash
sudo fuser -k 5100/tcp
```

或者换端口 5101。

------

## 5. `Ctrl+C` 停不下来

分布式训练有 8 个 rank，可能残留。直接杀：

```bash
pkill -f "prs.experiments.cli"
pkill -f "torchrun"
```

如果没权限：

```bash
sudo pkill -9 -f "prs.experiments.cli"
sudo pkill -9 -f "torchrun"
```

确认：

```bash
nvidia-smi
```
